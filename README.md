# つなげモン LP統合プラットフォーム (LP v2)

このリポジトリは、「つなげモン」シリーズのサービス紹介ランディングページ（プレビュー環境・本番コードエクスポート対応）のソースコードです。

本プロジェクトは **Astro (静的サイトジェネレーター)** と **Tailwind CSS** を用いて構築されており、JSONデータを編集するだけでサイト上のソリューション（テーマ）一覧を動的に生成・更新できる仕組みになっています。

---

## 1. データ駆動型のコンテンツ管理 (`theme.json`)

サイト上の「5つの連携ソリューション」セクションは、データファイルである `data/theme.json` から自動的に生成されます。
このファイルを編集することで、エンジニアでなくてもサイト上のテキストを簡単に変更することができます。

### ファイルの場所
`data/theme.json`

### JSONの書き方（スキーマ）

```json
{
  "themes": [
    {
      "id": "01",                   // テーマの番号 (例: "01"〜"07") ※デザインのカラーなどに影響します
      "title": "つなげモン ポータル",   // 見出しとして大きく表示されるサービス名
      "subtitle": "〜 取引先と「つなげモン」 〜", // その下のサブタイトル
      "description": "顧客、協力会社...<span class=\"...\">マイページ</span>を介して...", // 説明文 (HTMLタグが使用可能)
      "problems": [                 // 解決できる課題（チェックマーク付きのリストとして表示）
        "電話・FAXでの受発注",
        "転記ミス・納期確認の手間"
      ]
    }
  ]
}
```

### JSONデータがビルドにどう反映されるか
ビルドを実行すると、Astroの `web/src/components/ThemeCard.astro` が `theme.json` のデータを読み込み、以下のルールに従ってHTMLを生成します。

1. **カラーリング**:
   `id` の数値に基づいてテーマカラーが自動で割り当てられます。（例: "01"は青、"02"は緑、"06"はオレンジなど）
2. **NEWバッジ**:
   現状の仕様では、`id` が `"06"` 以上の項目には自動的に「NEW」という赤いバッジが付与されます。
3. **カスタムリンク（PDF等）**:
   `id` が特定の番号（"03", "06", "07"など）の場合のみ、専用の詳細ページや「提案書（PDF）をダウンロード」といった特別なボタンが表示されるように `ThemeCard.astro` 内で条件分岐されています。

> **💡 ヒント:** 
> 新しいテーマを増やす場合は、`theme.json` に新しいブロックを追加するだけで、レイアウトが崩れることなく新しいカードが生成されます。
> もし「新しいテーマにPDFダウンロードボタンを追加したい」場合は、`ThemeCard.astro` 内の条件分岐（`theme.id === "08"` など）を追加する必要があります。

## 2. プレビュー環境と本番用コードのエクスポート

このシステムは **デュアルモード（プレビューモード・本番モード）** に対応しています。

### プレビューサイトとしての運用
GitHubにPushすると、自動的にプレビュー環境（GitHub Pages等）が更新されます。
プレビューサイトの右下には **「管理者プレビューモード」のパネル（歯車アイコン）** が表示されます。

### 本番サイト（自社Webサーバー）への配置
プレビュー画面の「本番用コード (ZIP)」ボタンをクリックすると、`production-build.zip` がダウンロードされます。
このZIPファイルは、本番のWebサーバー（自社のホームページ等）にそのままアップロードして使うためのものです。

**ZIPファイルの特徴:**
1. **社外秘資料の完全除外**: プレビューサイトには存在する社内確認用の提案書PDFなど（`slides/` ディレクトリ等）は、ビルド時に自動で削除され、本番用ZIPには含まれません。
2. **完全相対パス**: CSSや画像などの読み込みパスがすべて完全相対パス（`./`）に変換されているため、自社サイトの「一番上の階層」に置いても「特定のフォルダの中」に置いても、絶対にリンク切れを起こさず完璧に表示されます。
3. **管理者パネルの非表示**: ZIP化される本番用コードからは、右下の管理者パネル自体が自動で消去されます。

## 3. 開発者向けコマンド

ローカルで手動ビルドやプレビューを行うためのコマンドです。

```bash
# 開発用ローカルサーバーの起動 (CSS等のリアルタイムプレビュー)
cd web
npm run dev

# 手動での完全ビルド (プレビュー用サイト生成 + 本番ZIP生成)
./build.sh

# 本番ZIPのみを手動で生成 (プレビューは無視)
./build_prod.sh
```
