# つなげモン LP統合プラットフォーム (LP v2)

このリポジトリは、「つなげモン」シリーズのサービス紹介ランディングページ（プレビュー環境・本番コードエクスポート対応）のソースコードです。

本プロジェクトは **Astro (静的サイトジェネレーター)** と **Tailwind CSS** を用いて構築されており、JSONデータを編集するだけでサイト上のソリューション（テーマ）一覧を動的に生成・更新できる仕組みになっています。

---

## 1. データ駆動型のコンテンツ管理 (Markdown / Content Collections)

サイト上の「5つの連携ソリューション」セクションは、**`web/src/content/themes/` 配下にある Markdown ファイル** から自動的に生成されます。
このファイルを編集するだけで、Webサイト（LP）と営業用PDF資料の両方が自動更新されます。

### ファイルの場所
`web/src/content/themes/` の中に各テーマの `.md` ファイルを作成します。

> **⚠️ 注意:**
> `data/theme.json` は、Typst（PDF生成）向けにビルド時に**自動生成**されるファイルです。
> `data/theme.json` を直接編集すると、次回のビルド時にMarkdownの内容で上書きされ消えてしまうため、**絶対に編集しないでください。**

### Markdownの書き方（スキーマ）

```markdown
---
id: "01"                   # テーマの番号 (カラーなどに影響)
title: "つなげモン ポータル"   # 見出し
subtitle: "〜 取引先と「つなげモン」 〜"
description: '顧客、協力会社...<span class="...">マイページ</span>を介して...'
problems:
  - 電話・FAXでの受発注
  - 転記ミス・納期確認の手間
dateAdded: "2026-07-21"    # (任意) 30日以内なら自動で「NEW」バッジ
image: "assets/サンプル/hoge.png" # (任意) カード右側に表示する固有画像 (svg/png等)
button:                    # (任意) ボタンを表示する場合
  type: "pdf"
  url: "slides/simulator_presentation.pdf"
  text: "提案書（PDF）をダウンロード"
  icon: "file-down"
---

ここから下には、将来的にWebサイト専用の詳細ページ（特化LP）となる長いテキストや事例などを自由に書くことができます。
（この文章はPDFには出力されないため、Web特化のSEOコンテンツとして活用できます）
```

### Markdownデータがビルドにどう反映されるか
ビルドを実行すると、Astroの `web/src/components/ThemeCard.astro` がMarkdownのメタデータ（Frontmatter）を読み込み、HTMLを生成します。

1. **カラーリング**:
   `id` の数値に基づいてテーマカラーが自動で割り当てられます。（例: "01"は青、"02"は緑など）
2. **画像の出し分け**:
   `image` が設定されていれば指定画像を表示し、なければ抽象的な連携図形をデフォルトで表示します。
3. **自動NEWバッジ機能 (完全自律型)**:
   `dateAdded` を指定すると、閲覧者のブラウザ上で現在の日付と比較し、**30日以内であれば自動的に「NEW」という赤いバッジを付与**します。
4. **PDF（Typst）への自動連携**:
   ビルド時、自動的に全Markdownのメタデータから `data/theme.json` が生成され、TypstはそのJSONを読み込んで美しいPDFの提案書を出力します。

## 2. プレビュー環境と本番用コードのエクスポート

このシステムは **デュアルモード（プレビューモード・本番モード）** に対応しています。

### プレビューサイトとしての運用
GitHubにPushすると、自動的にプレビュー環境（GitHub Pages等）が更新されます。
プレビューサイトの右下には **「管理者プレビューモード」のパネル（歯車アイコン）** が表示されます。

### 本番サイト（自社Webサーバー）への配置
プレビュー画面の「本番用コード (ZIP)」ボタンをクリックすると、`production-build.zip` がダウンロードされます。
このZIPファイルは、本番のWebサーバー（自社のホームページ等）にそのままアップロードして使うためのものです。

**ZIPファイルの特徴:**
1. **社外秘資料の完全除外**: プレビューサイトには存在する社内確認用の提案書PDFなど（`slides/` ディレクトリ等）は、ビルド時に自動で削除され、本番用ZIPには含まれません。
2. **完全相対パス**: CSSや画像などの読み込みパスがすべて完全相対パス（`./`）に変換されているため、自社サイトの「一番上の階層」に置いても「特定のフォルダの中」に置いても、絶対にリンク切れを起こさず完璧に表示されます。
3. **管理者パネルの非表示**: ZIP化される本番用コードからは、右下の管理者パネル自体が自動で消去されます。

## 3. 開発者向けコマンド

ローカルで手動ビルドやプレビューを行うためのコマンドです。

```bash
# 開発用ローカルサーバーの起動 (CSS等のリアルタイムプレビュー)
cd web
npm run dev

# 手動での完全ビルド (プレビュー用サイト生成 + 本番ZIP生成)
./build.sh

# 本番ZIPのみを手動で生成 (プレビューは無視)
./build_prod.sh
```
