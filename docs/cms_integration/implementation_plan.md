# 実装計画: CMSインテグレーション

## 背景と課題
MarkdownのFrontmatter (YAML) をデータソースとして用いる「つなげモン LPエンジン」において、非エンジニア（営業やマーケティング担当）がソースコードを直接触らずに更新できる環境が求められていた。当初 Keystatic を導入したが、Astro 5.x系での静的エクスポート (`output: static`) と SSR（サーバーサイドレンダリング）の混在がローカル環境でクラッシュを引き起こした。

## 解決策とアプローチ
- **KeystaticからDecap CMSへの切り替え**: バックエンドAPIを必要としない完全静的なSPAとして動く Decap CMS を採用。
- **ローカル中継サーバーの利用**: `npx decap-server` をローカルで立ち上げることで、GitHub APIなどを経由せずに直接ローカルのGitリポジトリのファイルを読み書きする手法を確立。
- **スキーマの完全同期**: `config.yml` 内で `BlockCards`, `BlockChatDemo` などLPで定義した全11種類のブロックを完全網羅。
- **UI最適化**: Astroコンポーネントがリアルタイムプレビューに非対応であるため、右側のプレビューペインを意図的に無効化し、フルスクリーンでのフォーム入力に最適化。
