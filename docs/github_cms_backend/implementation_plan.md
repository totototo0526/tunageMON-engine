# TunageMON LP v2 段階的開発ロードマップと実装計画

## 概要 (Overview)
本計画は、TunageMON LP v2 の開発を「既存の静的サイトビルドおよびローカル編集環境に影響を与えない最小限のステップ」で安全に推進するためのロードマップです。

最優先タスクである **Decap CMS の GitHub Backend 移行 (Plan A)** およびアセット整理・ビルド検証を、リスクの低いフェーズに分けて順次進めます。

---

## 開発ロードマップ (Roadmap)

### フェーズ 1: ドキュメント整理 & ロードマップ確定 【本フェーズ】
- **目的**: プロジェクト全体方針の明確化、安全な開発プロセスの確立
- **作業内容**:
  - `docs/github_cms_backend/` 配下にタスク・実装計画ドキュメントを生成
  - [`agent.md`](file:///media/ksp-zorin-001/WORK_DISK/workspace/tunageMON/lp_v2_work/agent.md) のステータス・タスクリスト更新
- **影響・リスク**: ゼロ（ドキュメントの追加・更新のみ）

---

### フェーズ 2: アセット整理とビルド同期検証 【低リスク】
- **目的**: 既存アセット・画像パスの整合性確認と Astro / Typst 両ビルドの安全確認
- **作業内容**:
  - `web/public/assets/` およびテーマ用画像のパス参照テスト
  - `build.sh` による Astro 静的サイトビルド ＆ Typst PDF 生成の動作検証
- **影響・リスク**: 既存コードの壊れを防ぐ回帰テスト中心

---

### フェーズ 3: Decap CMS GitHub Backend (Plan A) 準備 & 設定設計 【中リスク・分離検証】
- **目的**: チーム協調編集に向けた GitHub リポジトリ直接連携の設定
- **作業内容**:
  - `config.yml` の `backend` 設定設計（`name: github`, `repo: ...`, `branch: main`）
  - ローカル編集 (`local_backend: true`) との互換性維持
  - OAuth 認証ゲートウェイ（または OAuth プロキシ/Decap 認証サーバー）の選定・手順策定
- **影響・リスク**: CMSの設定変更のため、ローカル開発モードを保護した状態で段階テストを実施

---

### フェーズ 4: 動作検証 & チーム運用マニュアル整備 【低リスク】
- **目的**: 移行完了後のチーム運用プロセスの標準化
- **作業内容**:
  - 非エンジニアメンバー向けの編集・公開手順書の作成（`docs/` 配下）
  - `build_prod.sh` による本番出力の最終テスト

---

## 検証計画 (Verification Plan)

### Script Verification
- `build.sh` を実行し、Astro ビルドエラーがなく、Typst PDF が正常生成されることを確認。
