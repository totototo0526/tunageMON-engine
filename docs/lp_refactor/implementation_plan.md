# 実装計画: つなげモン LP リファクタリング

## 現状の課題
- 現在のLPはベタ書きのHTML（`LP.html` 等）で構成されている可能性が高い。
- 新しいセクションの追加やテキストの編集時に、全体の構造を壊してしまうリスクがあり、メンテナンスコストが高い。

## 解決策のアプローチ
追加や編集を容易にするため、コンテンツと構造（レイアウト）を分離し、コンポーネント化する仕組みを導入する。
ユーザーとの相談のもと、以下のいずれかのアプローチを採用する。

### アプローチ案
1. **PHP includeの活用 (シンプル構成)**
   - Header, Footer, Section ごとにPHPファイルに分割し、メインファイルで `include` する。
   - メリット: ビルド不要で既存のサーバー環境で動きやすい。
   - デメリット: テキストデータ自体の分離はされない。

2. **静的サイトジェネレーター (SSG: Astro 等) の導入**
   - コンポーネント指向 (UIパーツの共通化) とデータ指向 (MarkdownやJSONでのコンテンツ管理) を両立。
   - メリット: 運用保守が極めて楽になる。
   - デメリット: Node.js環境とビルド手順が必要。

3. **Vanilla JS + JSON データ駆動**
   - 骨組みのHTMLを用意し、テキストや画像パスなどのコンテンツを外部のJSONからJSで描画する。
   - メリット: ビルド不要、テキスト編集はJSONを触るだけ。
   - デメリット: JSが無効な環境やSEOに若干の懸念（※SSGほどの恩恵はない）。

## 今後のステップ
1. 現状のメインファイル（`LP.html` など）を確認し、コピーして作業環境 (`work_lp` 等のディレクトリ) を構築する。
2. ユーザーと最適な「仕組み」について合意する。
3. 選定した仕組みに沿って、コピーしたファイル群をリファクタリングする。
