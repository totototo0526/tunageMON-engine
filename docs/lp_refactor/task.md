# つなげモン LP リファクタリングタスク

## 目的
LPの追加・編集を容易にするため、保守性の高い仕組みへ移行する。直接既存ファイルを編集せず、コピーファイル・ディレクトリで作業および検証を行う。

## タスク一覧
- [ ] LPのメインファイル（`LP.html` または `index.html`）の特定
- [ ] 作業用ディレクトリ/ファイルの作成（オリジナルからのコピー）
- [ ] 追加・編集に強い仕組みの選定（ユーザーと合意）
  - 案1: PHPによるコンポーネント分割
  - 案2: SSG (Astro/Vite) 導入によるコンポーネント化・データ分離
  - 案3: Vanilla JS + JSON によるコンテンツの動的読み込み
- [ ] 選定した仕組みに基づくリファクタリングの実施
- [ ] 動作確認・デザイン崩れのチェック
- [ ] 本番ファイルへの反映（または本番公開手順の確立）
