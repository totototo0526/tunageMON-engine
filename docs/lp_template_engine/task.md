# Task: 特化LPテンプレートエンジンの構築 (LP Template Engine)

## 目的
原氏から受領した「つなげモン No.2」のような特化型ランディングページのデザインをAstroアーキテクチャに吸収し、ヘッダー・フッターのトンマナを統一しつつ、Markdown（MD）のみで多様なデザインレイアウトを展開可能にする。

## 主要な要件
1. **デザインコンポーネント化**: 受領したHTMLの各セクション（ヒーロー、悩み、解決策、導入シナリオ、料金など）をAstroコンポーネント（`.astro`）に分割・再利用可能にする。
2. **ヘッダー/フッターの統一**: 特化LP独自のヘッダー/フッターを廃止し、本プラットフォーム（ポータル）共通のヘッダー/フッターへ差し替える。
3. **Markdown（CMS）からのレイアウト制御**:
   - `02-theme.md` 等のFrontmatterで `layout: pattern_a` のように指定することで、デザインを切り替えられる仕組みを作る。
   - 本文（Body）や独自のFrontmatterフィールドを利用して、コンポーネントに流し込むコンテンツを動的に管理する。

## タスク一覧
- [ ] 受領したHTMLの解析と、Astroコンポーネントへの分割設計。
- [ ] 共通レイアウト（BaseLayout）の適用によるヘッダー/フッターの統一。
- [ ] `web/src/pages/themes/[slug].astro` （動的ルーティング）の構築。
- [ ] Markdownスキーマ（`content.config.ts`）の拡張（特化LP用のデータフィールド追加）。
- [ ] （オプション）将来的な複数テンプレート切り替え機能の実装検討。
