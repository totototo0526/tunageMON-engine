# Walkthrough: 特化LPテンプレートエンジンの構想と統合戦略

## 1. 背景
原氏より「つなげモン No.2」向けの優れた特化型ランディングページ（HTML）が共有されました。
これを現在のシステムにどう統合するか、チャットでのブレインストーミングを通じて以下の結論に至りました。

## 2. 議論のサマリー

### 課題の認識
1. **トンマナのズレ**: そのままHTMLを公開すると、ヘッダーやフッターの形状がメインのポータルサイトと異なってしまい、ユーザー体験（回遊性）が損なわれる。
2. **保守性の低下**: 毎回HTMLを手書きでコーディングする状態に戻ってしまい、せっかく構築した「MarkdownベースのCMS」の利点が活かせない。

### 決定事項とアプローチ
- **「デザインの吸収とMarkdown化」**: 
  特化LP内の素晴らしいコンテンツ構成（ヒーロー、悩み、解決策、導入シナリオなど）をAstroコンポーネントとして分解・再構築し、「つなげモンプラットフォームの標準デザイン」の一部として吸収する。
- **ヘッダー・フッターの統一**:
  LP固有のヘッダー・フッターは破棄し、プラットフォーム標準のものに差し替える。
- **マルチパターンへの布石**:
  将来的に「Markdownで同じ文章を書いても、`layout` プロパティを変えるだけでレイアウトがガラッと変わる」ような、本格的なテンプレートエンジンの基礎として、今回のHTMLを利用する。

## 3. システム構造の変化（予定）

### Before
```text
web/src/content/themes/*.md 
  ↓
[生成] index.html (ポータルのみ)
[手動] 個別HTMLを書いて都度アップロード
```

### After
```text
web/src/content/themes/*.md 
  ↓
[生成] index.html (ポータル)
[生成] themes/01/index.html (特化LP / テンプレートA適用)
[生成] themes/02/index.html (特化LP / テンプレートB適用)
```

これにより、コンテンツ（テキスト）とデザイン（レイアウト）が完全に分離され、非常に強力なマーケティング基盤が完成します。
