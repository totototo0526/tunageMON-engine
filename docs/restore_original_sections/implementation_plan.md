# 実装計画 (Implementation Plan)

## 背景と目的
前回構築した「Markdown駆動の動的テンプレートエンジン（`TemplateA`）」は、骨組み（ヒーロー、課題、料金）のみが反映されており、中間セクションがデモ用として省略されていました。
ユーザーからの「副官ではらさんのページ（原氏デザインのオリジナルLP）を再現して」という要望に応えるため、不足している全セクションを抽出し、テンプレート内に復元します。

## 対象ファイル
1. `web/src/components/lp-templates/TemplateA.astro`
   - 不足しているセクション群のHTMLを、既存の構成を破壊することなく挿入します。
2. （必要に応じ）画像パスの相対パス置換
   - 静的HTML内では `assets/...` となっていたパスを、Astro上の動的ルーティング（例：`/themes/07-theme/index.html`）に対応させるため、`../../assets/...` へと置換します。

## 実施ステップ
1. **不足セクションの特定**
   - 元HTML（`bira.html`など）から、解決策・導入シナリオ・比較表・セキュリティ・デメリット・FAQなどの`<section>`ブロックを特定します。
2. **`TemplateA.astro` の更新**
   - 省略用コメントとして残していた `<!-- ※デモ用のため... -->` の箇所をターゲットとして、スクリプト等を利用して抽出したHTMLを一括挿入します。
3. **パスの正規化**
   - HTML内の `<img src="assets/...">` を `<img src="../../assets/...">` に一括修正します。
4. **動作検証・証跡の保存**
   - `./build.sh` で本番ビルドの正常終了を確認します。
   - `npm run dev` サーバーを立ち上げ、Puppeteerによりローカルでのレンダリング結果をスクリーンショットおよび動画で記録し、ユーザーへの確認用資料（Walkthrough）として保存します。

## 今後の拡張（設計思想・Core Philosophyとの連携）
- 現状はHTMLを直接埋め込んでいますが、CMSの柔軟性を高めるため、今後はこれらの中間セクションも「動的コンポーネント」に切り出すことが推奨されます。
- 例えば、FAQの内容や比較表の価格設定などを `content.config.ts` のスキーマに追加し、各テーマのMarkdownから動的に流し込めるようにすれば、全テーマに特化したパーソナライズLPがさらに容易に生成可能になります。
