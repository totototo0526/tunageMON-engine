# サンプルコード（sampleA.html）のLPサイトへの統合計画

## 背景・目的
`assets/サンプル/sampleA.html` にある「軽貨物管理システムのモバイルシステム概要」のソースコードを、メインのランディングページ（`LP.html`）に統合します。
サンプルコードの図表は横幅が広く詳細な内容であるため、現状の「 Theme 03: つなげモン モバイル（50/50のレイアウト）」内にそのまま配置するとレイアウトが崩れたり視認性が低下する可能性があります。

そのため、以下のいずれかのアプローチで統合することを提案します。

## Open Questions

> [!IMPORTANT]
> 統合のレイアウト方針について、ご希望のアプローチを以下の3つからお選びください。

**案1: 新しい「ユースケース（導入事例）」セクションとして追加する（推奨）**
「5つの連携ソリューション」セクションの直下に、新しく「ユースケース：軽貨物管理システムの例」というフルワイドのセクションを作成し、サンプルの図表をそのまま組み込みます。視認性が最も高く、LPのストーリーとしても自然です。

**案2: Theme 03 を展開可能なアコーディオン/モーダルにする**
Theme 03 のカード内に「詳細な利用イメージを見る」というボタンを配置し、クリックするとサンプルの図表がモーダルウィンドウ（またはアコーディオン）で大きく表示されるようにします。LP全体の長さを抑えたい場合に有効です。

**案3: Theme 03 のレイアウト自体を縦並びに変更する**
Theme 03 だけ、左にテキスト・右に図という現在のレイアウトから、「上にテキスト・下にサンプルのフルワイド図表」というレイアウトに変更します。

## Proposed Changes

（※案1の場合の変更予定箇所）

### LP.html

#### [MODIFY] LP.html
- `sampleA.html` の `<style data-purpose="layout-and-colors">` の内容を `LP.html` に統合（または `assets/css/style.css` に追記）。
- `LP.html` のソリューションセクションの下に、新規の `<section id="usecase">` を追加。
- `sampleA.html` の `<main>` 内にあるダイアグラム（`<section data-purpose="diagram-container">`以下）をコピーし、新規セクション内に配置。
- フォントや背景色などの微調整を行い、全体のデザイン（Tailwind CSS）と馴染ませる。

## Verification Plan

### Manual Verification
- ローカル環境のブラウザで `LP.html` を開き、統合したサンプルの図表が正しく表示されること（レスポンシブ対応含む）を確認する。
- 既存の Theme 03 などの他のレイアウトに悪影響が出ていないか確認する。
